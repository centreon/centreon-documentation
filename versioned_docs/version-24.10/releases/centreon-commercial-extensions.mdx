## Introduction

You can find in this chapter all changelogs concerning **Centreon Commercial Extension**.

> It is very important when you update your system to refer to this section in order to learn about behavior changes or
> major changes that have been made on this version. This will let you know the impact of the installation of these
> versions on the features you use or the specific developments that you have built on your platform (modules,
> widgets, plugins).

If you have feature requests or want to report a bug, please contact support.

Read more about version 24.10 in our [blog post](https://www.centreon.com/en/blog/centreon-fall24-whats-new-in-the-24-10-software-version/).

## Centreon MAP

### 24.10.0

Release date: `October 30, 2024`

<details open>
  <summary>Enhancements</summary>

- [API] Fixed access control for some v2 API endpoints.
- [Editor] Added an option to hide the label for containers.
- [Editor] Increased the number of available fonts and make Roboto the default one.
- [GeoViews] Redesigned the edit modal.
- [Legacy] MAP Legacy is no longer supported.
- [Protobuf] Implemented "pb_instance_configuration" event.
- [SELinux] Packaged SELinux rules for centreon-map and centreon-mbi.
- [Server] Adapted Protobuf to handle the new event "AdaptiveHostStatus" from Broker.
- [Server] Adapted Protobuf to handle the new event "AdaptiveServiceStatus" from Broker.
- [Server] Improved logs related to Broker event processing and made event queue size configurable.
- [Server] Improved the parsing of Broker events on creation and deletion of resources.
- [Server] Increased the default log level to INFO.
- [Server] Optimized the propagation of a resource status change.
- [Viewer] You can now display a host name for a service.
- [Viewer] You can now display the status as background color for a resource in weather style.

</details>

<details>
  <summary>Bug fixes</summary>

- [Server] Fixed a synchronization issue that could cause resources to disappear from a map.
- [Server] Fixed concurrent access issues causing exceptions in the viewer.
- [Server] Fixed Servicegroup status when configured using Service Templates or Hostgroup Services.

</details>
